# 聚合点智能演示系统 (Webmap 互动版) V我：13681513675

这是一个基于 `ArcGIS Maps SDK for JavaScript (V4.30)` 构建的动态 WebMap 应用界面。该项目展示了如何通过 ArcGIS Online 的配置提取要素，并在加载大量数据点时，通过“智能聚合（Clustering）”和自定义互动逻辑来优化地图浏览体验。

## ✨ 核心特性

- **多模式聚合逻辑**：根据点位密集程度自动将记录归集到一个群组图标中。
- **平滑动画下钻**：在地图较低缩放级别时，点击聚合数字，会自动根据聚合包含的所有子点的空间覆盖范围（`extent`）或者动态设定的增量平滑放大地图视角。
- **防覆盖检测补偿**：针对某些极端的数据点完全坐标重合问题，程序内置了防卡死机制（`zoomIncrement` 兜底强行放大）。
- **保留原生样式**：原汁原味地保留了在 ArcGIS Online 的 Map Viewer 制图工具中配置的高级色彩渲染和字段排序。
- **自定义属性拓展注入**：
  - 对时间戳类型的字段自动转化为人性化的 `YYYY年MM月DD日 hh:MM`。
  - 动态格式化经纬度坐标，呈现为标准度分秒（如 `31°24'47.31"N, 116°8'5.31"E`）。
  - 在所有原生弹窗（Popup）底部统一注入明确带有蓝色高亮的精确位置坐标组件。

## ⚙️ 界面控制面板功能

本系统左下角悬浮有详尽的状态指示器及参数设置面板：
- **行为模式切换**：“智能模式”、“总是放大”或“总是显示信息”。
- **缩放和缓冲控制**：使用者可以通过数字滑块自行控制地图的“最大放大级别限制”、聚合放大时每一次前进的“缩放步长”，以及用于空间要素分析拾取的“缓冲距离”（单位：米）。

## 🚀 部署指南

本页面的所有代码集成在 `Webmap 智能版V3.html` （后建议重命名为 `index.html` 以便访问）单个文件中，不依赖额外的后端支持，任何支持静态文件托管的服务皆可运行它。

### 通过 GitHub Pages 零成本发布（推荐）

本系统极其适合通过 GitHub 的免费 Pages 服务搭建公共外链：
1. 建立一个公开（Public）的 GitHub 代码仓库。
2. 将包含本文档及 HTML 文件（强烈建议重命名为 `index.html`）全部上传到仓库根目录。
3. 进入该仓库右上角的 **"Settings"** -> 左边栏的 **"Pages"** 设置模块。
4. 在 **Build and deployment** 下拉菜单中选择 **"Static HTML"** 或直接通过 **Deploy from a branch** 选择主分支。
5. 等待页面右上角的 "Actions" 流水线构建完成，即可获得专属的外网访问地址。
   *(详细的全操作步骤可参考同目录下附带的 `github_pages_部署指南.md`)*

## 📝 二次开发说明

若您需要修改引用的在线地图底座，请打开 HTML 源码，找到 `WebMap` 对象的实例化位置：
```javascript
const webmap = new WebMap({
  portalItem: {
    id: "YOUR_ARCGIS_ONLINE_ITEM_ID" // 替换为您的新 WebMap ID
  }
});
```
如果要修改字段对应的中英文翻译列表，请修改 `fieldNameMap` 字典常量。
